# Юридические документы

Шаблоны юридических документов для Telegram-бота.

## Минимальный набор документов

Согласно законодательству РФ (152-ФЗ, ГК РФ, Закон о защите прав потребителей) для платного Telegram-бота необходимо **минимум 2 документа**:

| Документ | Обязательность | Файл шаблона |
|----------|---------------|--------------|
| **Политика конфиденциальности** | Обязательна (152-ФЗ) | `privacy_policy_template.md` |
| **Пользовательское соглашение (Оферта)** | Обязательна для платных услуг | `terms_of_service_template.md` |

## Как использовать шаблоны

### Шаг 1: Скопируйте шаблоны

Скопируйте содержимое шаблонов в Google Docs, Notion или на ваш сайт.

### Шаг 2: Заполните плейсхолдеры

Замените все значения в квадратных скобках `[...]` на реальные данные:

```
[НАЗВАНИЕ_БОТА]           → Unibot
[USERNAME_БОТА]           → @unibot_ai
[ИП/ООО/Самозанятый]      → ИП Иванов Иван Иванович
[ПОЛНОЕ_НАИМЕНОВАНИЕ]     → Иванов Иван Иванович
[ОГРНИП/ОГРН]             → 123456789012345
[ИНН]                     → 123456789012
[АДРЕС]                   → г. Москва, ул. Примерная, д. 1
[EMAIL]                   → support@example.com
[TELEGRAM_КОНТАКТ]        → @support_bot
[ДАТА]                    → 01.01.2026
[НАЗВАНИЕ_ПЛАТЁЖНОЙ_СИСТЕМЫ] → ЮKassa
[ССЫЛКА_НА_ПОЛИТИКУ]      → https://docs.google.com/...
[ССЫЛКА_НА_ОФЕРТУ]        → https://docs.google.com/...
[ПЕРИОД]                  → каждые 30 дней
[КОЛИЧЕСТВО_N] / [ЦЕНА_N] → Ваши тарифы
```

### Шаг 3: Опубликуйте документы

**Варианты размещения:**

1. **Google Docs** (рекомендуется для старта)
   - Создайте документ
   - Настройки доступа: «Все у кого есть ссылка»
   - Скопируйте ссылку

2. **Notion**
   - Создайте страницу
   - Share → «Anyone with the link»
   - Скопируйте ссылку

3. **GitHub Pages** (для продвинутых)
   - Создайте репозиторий
   - Включите GitHub Pages
   - Разместите HTML/Markdown файлы

4. **Ваш сайт**
   - Разместите документы на отдельных страницах
   - Убедитесь в доступности 24/7

### Шаг 4: Добавьте ссылки в конфиг бота

Откройте `config.yaml` и добавьте ссылки в раздел `legal`:

```yaml
legal:
  enabled: true
  version: "1.0"
  privacy_policy_url: "https://docs.google.com/document/d/XXXXX/view"
  terms_of_service_url: "https://docs.google.com/document/d/YYYYY/view"
```

## Важные замечания

### Согласие пользователей

Бот автоматически запрашивает согласие при первом использовании:
- Показывает ссылки на документы
- Требует нажать «Принимаю» для продолжения
- Сохраняет факт согласия в БД

### Версионирование

При изменении документов:
1. Увеличьте версию (`version: "1.1"`)
2. Бот запросит повторное согласие у пользователей
3. Обновите дату в документах

### Уведомление Роскомнадзора

Если вы ИП или ООО и обрабатываете персональные данные автоматизированно:
1. Подайте уведомление через [Госуслуги](https://www.gosuslugi.ru)
2. Или через [портал Роскомнадзора](https://pd.rkn.gov.ru)

### Для иностранных пользователей

Если бот будет использоваться за пределами РФ:
- Добавьте английскую версию документов
- Укажите в Политике раздел про GDPR (уже включён в шаблон)

## Юридическая консультация

Шаблоны составлены на основе типовых документов и требований законодательства. Рекомендуется:
- Проконсультироваться с юристом перед запуском
- Адаптировать документы под специфику вашего бизнеса
- Регулярно проверять актуальность документов

## Дополнительные документы (опционально)

При необходимости можно добавить:
- **Согласие на рекламную рассылку** — если планируете отправлять рекламу
- **Политика cookie** — не актуально для Telegram-ботов
- **Соглашение об уровне обслуживания (SLA)** — для B2B клиентов
